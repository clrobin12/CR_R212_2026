# Rapport — TD2

## Points clés

### 1. Importance d’un HTML propre

J’ai compris que le CSS ne sert pas à corriger un mauvais HTML, mais à améliorer une structure déjà claire. Le HTML doit déjà être logique et lisible sans style, et le CSS vient seulement ajouter de la mise en forme. Ça permet d’avoir un code plus propre et plus facile à maintenir.

### 2. CSS minimal et lisibilité

J’ai appris qu’on peut rendre une page agréable avec peu de CSS. En travaillant sur la typographie, les espacements et la largeur du texte, on peut déjà obtenir un rendu lisible sans utiliser de layout complexe comme Flexbox ou Grid.

### 3. Layout avec Flexbox

J’ai utilisé Flexbox pour organiser certains éléments comme la navigation ou les sections avec plusieurs articles. J’ai compris qu’il fallait respecter l’ordre du HTML et éviter de “forcer” le layout avec des techniques compliquées, pour garder un site responsive et robuste.

## Utilisation de l'IA

### Démarche

J’ai utilisé l’IA uniquement lorsque je rencontrais des problèmes en CSS, notamment pour comprendre certaines propriétés ou corriger des erreurs dans mon code.

### Apports et limites

L’IA m’a aidé à débloquer certaines situations rapidement et à mieux comprendre certaines propriétés CSS. Mais j’ai dû adapter les solutions proposées à mon code pour qu’elles restent simples et respectent les consignes du TD.

### Prompts significatifs

1. "Pourquoi mon flexbox ne fonctionne pas ?" → m’a aidé à comprendre une erreur dans mon code  
2. "Comment centrer des éléments avec flexbox ?" → m’a permis d’améliorer mon layout
