# Rapport — TD3

## Points clés

### 1. Modèle événement → état → rendu

J’ai compris que toutes les interactions en JavaScript suivent un fonctionnement en trois étapes : d’abord un événement (comme un clic ou une saisie), ensuite un état qui change (par exemple un menu ouvert ou un compteur qui augmente), et enfin un rendu, c’est-à-dire ce qui change visuellement à l’écran. Par exemple, quand on clique sur un bouton pour afficher un menu, l’état passe à “ouvert” et une classe CSS est ajoutée pour afficher le menu.

### 2. classList.toggle : lien entre JS et CSS

La méthode classList.toggle permet d’ajouter ou retirer une classe automatiquement sur un élément. Elle est très importante car elle fait le lien entre le JavaScript et le CSS : le JavaScript gère le moment où la classe change, et le CSS s’occupe de l’apparence. Par exemple, on peut activer ou désactiver un mode sombre simplement en ajoutant ou retirant une classe.

### 3. Rôle du JavaScript dans une page Web

J’ai compris que le JavaScript sert à rendre les pages interactives. Contrairement au HTML qui structure la page et au CSS qui gère le style, le JavaScript permet de réagir aux actions de l’utilisateur, modifier le contenu et créer des comportements dynamiques sans recharger la page.
