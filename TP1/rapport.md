# Rapport — TP1

## Points clés

### 1. Utilisation de Flexbox et Grid

J’ai appris à utiliser Flexbox et Grid dans une même page, chacun pour des usages différents. J’ai utilisé Flexbox pour organiser des éléments sur une seule ligne comme la navigation ou le header, et Grid pour créer une mise en page plus structurée comme les cartes projets. J’ai compris qu’il faut choisir l’outil adapté selon le besoin.

### 2. Responsive et adaptation des écrans

J’ai compris comment rendre une page responsive en utilisant principalement Grid (avec auto-fit et minmax) et une media query pour les petits écrans. Le but est que la page reste lisible sur mobile sans devoir tout refaire. J’ai testé en réduisant la fenêtre pour vérifier que tout s’adapte correctement.

### 3. Utilisation des variables CSS

J’ai utilisé des variables CSS pour gérer les couleurs, les espacements et la typographie. Cela permet de garder un code plus propre et plus facile à modifier, car on peut changer une valeur à un seul endroit au lieu de la répéter partout.

## Utilisation de l'IA

### Démarche

J’ai utilisé l’IA lorsque je rencontrais des problèmes, notamment pour comprendre pourquoi mon fichier style.css n’était pas bien lié à mon fichier index.html.

### Apports et limites

L’IA m’a aidé à identifier des erreurs comme un mauvais chemin dans le lien du fichier CSS ou une faute dans la balise `<link>`. Cela m’a permis de corriger rapidement le problème. 

### Prompts significatifs

1. "Pourquoi mon fichier CSS ne s’applique pas à ma page HTML ?" → m’a aidé à vérifier le chemin et la balise link  
2. "Comment bien relier un fichier CSS à un fichier HTML ?" → m’a permis de comprendre la bonne syntaxe
